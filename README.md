# AI & AR Lab

Цифровая образовательная платформа по структуре авторской программы Анны Валерьевны Колтуновой **«Применение искусственного интеллекта и дополненной реальности в проектной деятельности»**.

## Возможности платформы

- 34 интерактивных урока в исходной последовательности программы и шесть модулей общей продолжительностью 34 часа.
- Единая data-driven страница урока: цель, теория, проектный маршрут, встроенная презентация и практическая работа.
- Словарь новых слов с полем для собственного объяснения ученика.
- Тестирование с объяснением ответов, повторным прохождением и сохранением лучшего результата.
- Рефлексия «Асык» с четырьмя состояниями понимания и текстовым комментарием.
- Портфолио ученика: для каждого урока сохраняются название, описание и ссылка на работу.
- Общий прогресс по 34 урокам, восстановление последнего урока и перенос данных первой версии.
- Адаптивный интерфейс, поддержка safe area мобильных устройств и базовый офлайн-режим PWA.

## Методические материалы

В репозитории на момент разработки **не было файлов КСП, авторских презентаций или раздаточных материалов**. Поэтому платформа:

1. строго сохраняет предоставленные названия, порядок, модули и часы программы;
2. использует нейтральные учебные сценарии, сформированные по теме и практическому продукту каждого урока;
3. не выдаёт автоматически созданные тексты за дословные материалы автора;
4. готова к подключению утверждённых КСП и презентаций через `js/course-data.js` без изменения страниц и логики платформы.

Перед публикацией в качестве официальной версии курса содержание рекомендуется сверить с авторскими КСП и заменить/дополнить записи данных утверждёнными формулировками.

## Добавление авторских материалов

Каждая запись `courseData.lessons` содержит:

- номер и модуль;
- название урока;
- цель;
- практический продукт;
- новые термины.

Страница открывается по адресу `lesson.html?lesson=НОМЕР`, а презентация — `presentation.html?lesson=НОМЕР`. После добавления содержания в запись урока его можно вывести в общий шаблон без создания отдельного HTML-файла.

## Локальный запуск

Сборка и установка зависимостей не нужны:

```bash
python3 -m http.server 8080
```

Откройте [http://localhost:8080](http://localhost:8080). Для service worker требуется HTTP/HTTPS; режим `file://` не подходит.

## Публикация на Netlify

- **Build command:** оставить пустым.
- **Publish directory:** `.`

Проект статический, не использует платные API, авторизацию и серверную передачу ученических данных.

## Хранение данных

Прогресс, практика, словарь, рефлексия, результаты тестов и портфолио сохраняются только в `localStorage` текущего браузера под ключом `aiArCourseProgressV2`.
